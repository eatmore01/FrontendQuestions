- батчинг - это процесс когда несколько вызовов setState схлопываются в один

```typescript
const [state, setState] = useState(0)

const handler = () => {
  setState(state + 1)
  setState(state + 1)
  setState(state + 1)
  setState(state + 1)
  setState(state + 1)
  setState(state + 1)
}

// при отработки хендлера стейт станет больше на 1 из за батчинга
// но если писать (prevState => prevState + 1) - то тут при отработке
// счетчик увеличится на то кол-во сколько было таких сет стейтов,
// я понимаю что жто происходит так, потому что когда ты не берешь превалью,
// у тебя функиц ябудет отрабатыать как 0 + 1 6 раз, а когда с прошлым значением то стейт создает очередь из вычеслений и ..так считает
```
